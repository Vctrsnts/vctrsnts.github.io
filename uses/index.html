uses

2026-06-26 · 2 min

Esta es la configuración y las herramientas que utilizo actualmente en mi día a día.

La página está inspirada en la idea de /uses, habitual en la IndieWeb y la smol web: compartir el hardware, software y entorno con el que uno trabaja y escribe.

Nada especialmente exótico. Solo herramientas con las que me siento cómodo y que intento mantener simples y duraderas.

▸ hardware principal

Lenovo ThinkCentre M90q Gen 3

  • CPU: Intel Core i5-12500
  • RAM: 32 GB
  • GPU: AMD Radeon Pro WX 4100
    • Aunque actualmente quiero cambiarla por una AMD Radeon Pro W6400
  • Monitor: Dell P2419H 24" — 1920×1080 @ 60 Hz

Equipo principal para:

  • Programación
  • Escritura
  • Virtualización ligera
  • Self-hosting
  • Uso diario general

▸ sistema

  • Distribución: Arch Linux x86_64
  • Kernel: Linux LTS
  • Sesión gráfica: Wayland
  • Gestor de ventanas: Hyprland
  • Tema GTK/colores: Nordic
  • Iconos: Nordzy-dark

Intento mantener un entorno relativamente minimalista, centrado en teclado, terminal y aplicaciones ligeras.

▸ terminal

  • Terminal: Kitty
  • Shell: Bash
  • Prompt: minimal
  • Fuente: JetBrains Mono 10pt

La mayor parte del tiempo trabajo desde terminal: SSH, administración de sistemas, edición rápida y automatización simple.

▸ software habitual

  • Editor de texto: Geany
  • Cliente de correo: Claws Mail
  • Virtualización: QEMU/KVM
  • Gestor de archivos: Midnight Commander / Thunar
  • Navegador principal: Firefox
  • Contenedores: Docker

Intento priorizar aplicaciones:

  • Rápidas
  • Simples
  • Mantenibles
  • Con pocas dependencias
  • Que funcionen bien en Linux

▸ servidor local

Lenovo ThinkCentre M910x

Servidor local utilizado para:

  • Self-hosting
  • Contenedores Docker
  • Backups
  • Pruebas
  • Automatización

Especificaciones

  • CPU: Intel® Xeon® E-2286M CPU @ 2.40GHz
  • RAM: 32 GiB DDR4
  • Sistema operativo: unRaid 7.3.0
  • Cómo convertí un ThinkCentre M910x en mi servidor local

▸ móvil

Google Pixel 8 Pro

Elegido principalmente por:

  • Soporte a largo plazo
  • Android relativamente limpio
  • Compatibilidad con aplicaciones libres
  • Flexibilidad del ecosistema

▸ self-hosting

La mayoría de servicios personales corren en Docker sobre un servidor local.

Intento autoalojar únicamente aquello que:

  • Realmente utilizo
  • Puedo mantener sin complicarme
  • No depender críticamente de terceros

Algunos servicios habituales:

  • Backups
  • Monitorización
  • Feeds RSS
  • Analíticas
  • Sincronización
  • Herramientas internas

▸ filosofía del setup

Intento mantener un entorno:

  • Funcional
  • Simple
  • Silencioso
  • Relativamente minimalista
  • Fácil de mantener a largo plazo

Prefiero herramientas pequeñas y predecibles antes que software sobrecargado o excesivamente complejo.
