Usando Podman - La base de todo el sistema

2026-06-01 · 5 min · @Self-Hosting · #Podman

Nuevo artículo sobre mi migración de Docker a Podman.

Como ya expliqué en el primer artículo, os comentaba el porqué de la migración a Podman. En este nuevo artículo, os explicaré los containers que para mí son los más importantes y que, siguiendo el consejo de atareao, es recomendable instalarlos a través de quadlets, para que así toda la gestión se haga a través de systemd y, con ello, aprovechar las ventajas que trae consigo.

◇ Ventajas de usar quadlet

Aquí cada uno podrá ver las ventajas o desventajas con respecto a usar quadlets, pero para mí las ventajas son las siguientes:

  • Te apoyas en systemd, con lo que consigues que todo lo gestione el sistema:
    • logs
    • permisos
    • servicios
  • Los containers funcionan usando los permisos del usuario que los ha lanzado mediante systemctl --user.
  • Se inician en el mismo momento en que se inicia el sistema.
  • La base de conocimiento ya la tienes porque se maneja con herramientas estándar del propio sistema.

Seguramente también tendrá su parte negativa, pero para mí la que tiene más peso es la parte positiva 😁

Después de comentar el motivo de por qué voy a usar quadlets en vez de otro sistema, ahora paso a la explicación de los que, para mí, son los containers que tienen que ser la base sobre la cual construyas todo tu sistema (en mi caso, mi sistema).

◇ Instalación y configuración de los containers base

Nota

Aclarar de nuevo que esta información la podéis obtener, con un aspecto más técnico y más profesional, en la web de atareao y una explicación más detallada aquí de todo este proceso.

Después de esta pequeña anotación, ahora pasamos a los que, para mí, son los containers más importantes:

  • Traefik: Para la gestión de todos los containers que van a salir al exterior.
  • Crowdsec: Una capa más de protección para nuestro servidor y que ayudará a Traefik a hacer frente a esos ataques del mundo exterior.
  • PocketID: Barrera de acceso a nuestros servicios y que te obliga a identificarte.
    • Solo he conseguido que funcione bien con Traefik y con Beszel, que me lo dio a conocer davidochobits.
  • Dockge: Gestión del resto de containers (creación, actualización, etc.), aprovechando la potencia que nos vendrá. Podéis encontrar más información aquí, donde atareao explica los motivos de esta elección.

Recordad, tal como habíamos dicho en el anterior artículo, que la creación y gestión de todo lo relacionado con Podman se realizará a través del usuario apps.

Instalación y configuración de Traefik

Aquí tenéis los archivos que estoy usando para la creación y configuración de Traefik (los tenéis disponibles en mi repositorio de Codeberg):

~/.config/containers/available/
└── traefik/
    ├── proxy.network          # creación de la red con acceso al exterior
    ├── traefik-certs.volume   # volumen donde se almacenan los certificados
    ├── traefik-plugins.volume # volumen donde se almacenan los plugins
    └── traefik.container      # configuración completa del container Traefik

A continuación podéis ver los archivos de configuración de Traefik.

~/.config/traefik/
├── traefik.yml                   # archivo de configuración de Traefik
└── dynamic/                      # configuración dinámica
    ├── crowdsec.yml              # integración Crowdsec
    ├── headers.yml               # cabeceras HTTP
    ├── geoBlock.yml              # bloqueo geográfico
    └── dashboard-auth.yml.jinja  # plantilla Jinja

Instalación y configuración de Crowdsec

Aquí tenéis los archivos que estoy usando para la creación y configuración de Crowdsec (los tenéis disponibles en mi repositorio de Codeberg):

~/.config/containers/available/crowdsec/
├── crowdsec.container # configuració completa del container Crowdsec
└── crowdsec.volume    # volumen donde se almacena crowdsec

Los archivos de configuración de Crowdsec.

~/.config/crowdsec/
└── acquis.yaml    # configuración de logs para Crowdsec (formato Traefik)

Como podéis ver, si revisais el archivo acquis.yaml os dais cuenta de que se necesitará la obtención de un identificador que nos facilitará Crowdsec para realizar la conexión con Traefik (en este artículo podéis encontrar cómo lo hice para Docker), pero lo vuelvo a poner aquí porque así me sirve para mí mismo en un futuro :) Recordad que ahora estamos trabajando en Podman.

Obtener API Key de Crowdsec

# En mi caso, le pongo el nombre de traefik, vosotros podeis poner otro para identificarlo. 
# Normalmente se usa bouncer-traefik
apps@debian:~$ podman exec crowdsec cscli bouncers add traefik

Nota

Esto nos da un identificador (nuestro código API) y solo lo visualizamos una única vez. Luego ya no se puede obtener, así que apuntadlo.

Con este código, se añade a nuestro archivo acquis.yaml y ya tenemos nuestro container de Crowdsec en funcionamiento. Eso sí, recomiendo un reinicio de Traefik y Crowdsec.

Nota

Recordad que tanto Traefik como Crowdsec están funcionando como quadlets, así que para reiniciarlos tenemos que hacer:

apps@debian:~$ systemctl --user restart traefik crowdsec

Como añadido, os aconsejo que una vez tenemos instalado Crowdsec, añadir las COLLECTIONS (yo lo entiendo como actualizaciones de seguridad que ha de controlar). En mi caso, he añadido las siguientes:

  • crowdsecurity/linux para los posibles ataques que pueda tener GNU/Linux
  • crowdsecurity/traefik para los posibles bugs de Traefik
  • crowdsecurity/sshd para los posibles ataques de SSH
  • crowdsecurity/base-http-scenarios para los posibles ataques genericos para todo el tráfico HTTP
  • crowdsecurity/http-cve para los posibles ataques orientados a CVEs de web conocidas.

Para proceder a su instalación se hace de la siguiente manera:

apps@debian:~$ podman exec -it crowdsec cscli collections install crowdsecurity/traefik
apps@debian:~$ podman exec -it crowdsec cscli collections install crowdsecurity/sshd
apps@debian:~$ podman exec -it crowdsec cscli collections install crowdsecurity/linux
apps@debian:~$ podman exec -it crowdsec cscli collections install crowdsecurity/base-http-scenarios
apps@debian:~$ podman exec -it crowdsec cscli collections install crowdsecurity/http-cve

Lo único que hay que tener presente es indicar que Crowdsec pueda leer los logs de Traefik; recordad que esto lo hemos configurado aquí:

# traefik.yml
accessLog:
  filePath: "/var/log/traefik/access.log"
  format: json
  bufferingSize: 100
  fields:
    defaultMode: keep
    names:
      ClientUsername: keep
    headers:
      defaultMode: keep
      names:
        Content-Type: keep
        X-Forwarded-For: keep
  filters:
    statusCodes:
      - "300-302"
      - "400-409"
    retryAttempts: true
    minDuration: "10ms"

# crowdsec - acquis.yml
source: file
filenames:
  - /var/log/traefik/access.log
labels:
  type: traefik
