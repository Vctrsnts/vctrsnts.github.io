RSS-Funnel - Añadiendo mejoras

2026-06-08 · 3 min · @Self-Hosting · #VPS #Podman

Si recordáis, ya había escrito otro artículo donde hablaba de las ventajas de este container. Y, después de dos años, tengo que decir que lo sigo usando porque no he encontrado nada mejor y porque su consumo es ínfimo.

Como consecuencia de la migración, le di una vuelta de tuerca a la configuración que tenía para ver si podía mejorarla en algo.

Así que, buscando entre los issues del repositorio —donde siempre se pueden encontrar nuevas ideas—, encontré este issue que había abierto uGeek, donde le hacía varias preguntas al creador. En una de las respuestas explica cómo añadir una imagen al feed que se genera.

Otra de las mejoras que quería conseguir era obtener la imagen del episodio y así tenerla en el podcatcher para saber qué es lo que estoy escuchando. Pero tengo que decir que, para esto, sí tuve que tirar de IA, porque no encontré ningún issue que hablara del tema.

Lo conseguí y, de momento, ya tengo la configuración que más me interesa y a la que se le puede sacar más partido. Os la dejo a continuación por si a alguien le puede servir de ayuda:

auth:
  username: user
  password: 'password'

endpoints:
  - path: /rss.xml
    source:
      format: rss
      title: El meu feed personal.
      description: Els Podcast que segueixo. Entre ells hi son, Atareao, uGeek, FrenteAlCliente,PapaFriki, SubeParriba, DesahogoGeek, etc...
    filters:
      - merge:
          source:
            # PODCAST ATAREAO
            - https://anchor.fm/s/5a5b39c/podcast/rss
            # PODCAST DESAHOGO GEEK
            - https://anchor.fm/s/1218850/podcast/rss
            # PODCAST FRENTE AL CLIENTE
            - https://media.rss.com/frente-al-cliente/feed.xml
            # PODCAST PAPAFRIKI
            - http://feeds.feedburner.com/papafriki
            # PODCAST PODCASTLINUx
            # - https://podcastlinux.com/feed
            # PODCAST SUBE PARRIBA
            - https://anchor.fm/s/baa8920/podcast/rss
            - https://anchor.fm/s/fe699890/podcast/rss
            - https://prestocast.com/yoyo308/feed
            # PODCAST UGEEK
            - https://ugeek.github.io/podcast.xml
            # PODCAST SAYONARA TECNO
            - https://www.ivoox.com/feed_fg_f12083422_filtro_1.xml
            # PODCAST PA TI LA PERRA GORDA
            - https://www.ivoox.com/feed_fg_f11672823_filtro_1.xml
            # PODCAST HARDWARE ADDICTOS
            - https://www.ivoox.com/feed_fg_f1306079_filtro_1.xml
          filters:
            - modify_feed: feed.items = feed.items.slice(0, 100);
      - modify_feed: |
          feed.image = {
            url: "https://vctrsnts.codeberg.page/logo.jpg",
            title: "El meu feed personal",
            link: "https://vctrsnts.codeberg.page"
          }
      - modify_post: |
          post.author = null;
          post.description = null;
          post.content = null;
          post.categories = [];
          post.comments = null;
          post.link = null;
          post.source = null;
          post.extensions = {};
          post.itunes_ext = {
            image: post.itunes_ext?.image
          };
          post.namespaces = {};
          post.dublin_core_ext = null;
      - limit: 365d

Aunque no creo que mucha gente utilice esta aplicación, yo os la recomiendo porque podéis tenerla en vuestro VPS o servidor local y así, en vuestro podcatcher, evitar tener miles de suscripciones y concentrarlo todo en una sola suscripción.

La única pega es la funcionalidad de limitación de 365 días que incluí:

      - limit: 365d

La duda que me queda es qué hay que hacer para que esta limitación funcione correctamente: si se aplica automáticamente cada vez que se actualizan los feeds que sigues o si solo se ejecuta cuando reinicias el container.

Es la única duda que me queda. Por lo demás, funciona a las mil maravillas.
