◇ Cómo funciona NTFY

Antes de empezar con usuarios, permisos y configuraciones, primero creo que es importante entender cómo funciona todo el conjunto.

A mi modo de ver, los topics son los buzones donde se envían las notificaciones y solamente quien tiene la llave para abrir ese buzón puede enviar o leer los mensajes.

Es un punto que me costó pillar, pero una vez lo entiendes, el resto resulta mucho más sencillo.

Flujo completo de funcionamiento

Una explicación visual del proceso podría ser la siguiente (el ejemplo se explica en base al container de FlexGet):

Flexget
     ↓
Token BOT
     ↓
Usuario BOT
     ↓
Permisos escritura:
- podcast-sync
     ↓
Topic podcast-sync
     ↓
Usuario USER suscrito desde móvil
     ↓
Notificación recibida
  • FlexGet genera una acción que ejecuta el script sendNtfy.sh (lo podeis encontrar al final de este articulo).
  • Gracias al token y a los permisos de escritura del usuario bot, se puede enviar la notificación.
  • La notificación llega al topic podcast-sync.
  • Como el usuario user está suscrito al topic podcast-sync y además tiene permisos de lectura, recibe la notificación y la puede visualizar.

Con este esquema ya podemos entender mejor el resto de la configuración.

◇ Definición de usuarios

Habíamos dejado el container listo para su configuración y lo primero es la creación de los usuarios (incluido el administrador), para su posterior uso.

En mi caso, he creado tres usuarios:

  • Usuario administrador: será el encargado del mantenimiento de la configuración. Es recomendable ponerle una contraseña suficientemente segura para no tener problemas futuros.
  • Usuario normal: será el encargado de la recepción de las notificaciones. Estará suscrito a todos los topics para su visualización.
  • Usuario bot: será el encargado de enviar las notificaciones a los diferentes topics.

Si tuviéramos que mostrar una estructura sobre mi configuración, sería la siguiente:

Usuario administrador

administrador
├─ Contraseña larga y única
├─ Guardada en gestor de contraseñas
├─ Sin usar en móvil
├─ Sin usar en automatizaciones
├─ Sin tokens persistentes
├─ Sin suscripciones habituales
└─ Solo mantenimiento

Este usuario:

  • Crea usuarios
  • Modifica permisos
  • Gestiona incidencias
  • Su uso se debe reducir al máximo para no exponerlo.

Usuario normal

user
├─ Contraseña fuerte
├─ Usado en móvil y web
├─ Permisos de lectura
├─ Tokens solo si fueran necesarios
└─ Suscripciones a topics

Este usuario será el que:

  • Inicia sesión desde el móvil
  • Utiliza la interfaz web
  • Recibe las notificaciones

Usuario bot

bot
├─ No usado manualmente
├─ Tokens permanentes
└─ Escritura en topics

◇ Creación de usuarios

El procedimiento de creación de usuarios es bastante sencillo. Únicamente accedemos a la consola (bash) del container y es desde ahí donde los crearemos.

apps@debian ~> podman exec -it ntfy sh

# creación del usuario administrador
/# ntfy user add --role=admin administrador
/# enter password:
/# repeat password:

# creación de usuario normal
/# ntfy user add user
/# enter password:
/# repeat password:

Con esto ya hemos creado nuestro administrador y nuestro usuario. El proceso sería exactamente el mismo para cualquier otro usuario.

A tener en consideración

Antes de crear los usuarios, tenemos que descomentar estas líneas de nuestro compose, que habíamos dejado comentadas para verificar que todo funcionaba correctamente:

# NTFY_AUTH_FILE: /etc/ntfy/auth.db
# NTFY_AUTH_DEFAULT_ACCESS: deny-all
# NTFY_ENABLE_LOGIN: true
# NTFY_REQUIRE_LOGIN: true

Una vez creados los usuarios, reiniciamos el container y, a partir de este momento, siempre tendremos que autenticarnos para acceder a la web.

◇ Definición de topics

Nota

Lo que sí os aconsejo es tener definidos de antemano los topics que vayáis a necesitar para tener una base sólida.

También es recomendable que los topics se creen siempre en minúsculas para facilitar su gestión posterior.

En mi caso, los topics que voy a utilizar son:

  • podcast-sync: encargado de recibir todas las notificaciones relacionadas con los podcast que sigo.
  • unRaid: uno de los motivos principales por los que me pasé a NTFY. Recibe todas las notificaciones de mi servidor local.
  • media-transfer: encargado de notificarme todas las gestiones relacionadas con películas o series.

◇ Creación de topics

Para la creación de los topics, en principio no es necesario crearlos explícitamente.

Los métodos para crear los topics pueden ser:

  • Enviar una notificación expresamente al topic
  • Suscribirse al topic

Entonces, con el usuario normal, simplemente te suscribes a los topics y estos se crean automáticamente.

◇ Permisos

Una vez ya tenemos los topics creados, necesitamos dar permisos a los usuarios para que puedan utilizarlos y aquí volvemos a acceder a la consola de NTFY.

apps@debian ~> podman exec -it ntfy sh

# permisos lectura (read-only) para user

/# ntfy access user "podcast-sync" ro
/# ntfy access user "unRaid" ro
/# ntfy access user "media-transfer" ro

# permisos escritura (write-only) para bot

/# ntfy access bot "podcast-sync" wo
/# ntfy access bot "unRaid" wo
/# ntfy access bot "media-transfer" wo

Ahora nos queda la parte más importante: los tokens.

◇ Definición de tokens

Siguiendo con la analogía de los buzones, los tokens son las llaves que nos permiten abrir esos buzones para entregar o recoger notificaciones.

Creación de tokens

Tenemos que acceder a:

Account > Access tokens

Después pulsamos sobre Create Access Token y rellenamos:

  • Label: identificador que nosotros le vamos a dar. Es únicamente informativo.
  • Token Expiration: indicamos cuándo finaliza ese token.

En mi caso, nunca expiran porque son tokens que utilizará el usuario bot para enviar notificaciones.

Una vez creado el token, ya podremos enviar notificaciones a los diferentes topics.

◇ Envío de notificaciones

En mi caso, utilizaré bash y curl, porque todo se realizará desde dentro del sistema y, más concretamente, desde un container.

Me he creado un pequeño script en bash que me simplifica bastante el trabajo.

Nota

Este script lo utilizo para el envío de notificaciones desde el container de FlexGet, pero podéis utilizarlo como base para vuestras propias necesidades.

Script de envío — sendNtfy.sh

#!/bin/bash
# Us: ./sendNtfy.sh "task" "autor" "titol" "contentSize"

SCRIPT_DIR="$(cd "$(dirname "$0")" && pwd)"

taskName="$1"
autor="${2:-}"
titol="$3"
contentSize="${4:-}"
ntfyServer="${ntfyServer:-https://ntfy.sh}"

if [[ -z "$taskName" || -z "$titol" || -z "$ntfyToken" ]]; then
  echo "$(date '+%Y-%m-%d %H:%M:%S') [ERROR] Falten parametres o ntfyToken no definit" >> "${SCRIPT_DIR}/sendNtfy.log"
  exit 1
fi

# ===================== TAGS =====================
getTags() {
  case "$1" in
    downloadPodcast*) echo "headphones"    ;;
    moveMediaMovies)  echo "clapper"       ;;
    moveMediaSeries)  echo "tv"            ;;
    moveBooks)        echo "book"          ;;
    moveMusic)        echo "musical_note"  ;;
    *)                echo "warning"       ;;
  esac
}

tags="$(getTags "$taskName")"

# ===================== TOPICS =====================
getTopic() {
  case "$1" in
    moveMediaMovies|moveMediaSeries)
      echo "${topicMedia}"
      ;;
    downloadPodcast*)
      echo "${topicPodcast}"
      ;;
    *)
      echo "${ntfyTopicDefault}"
      ;;
  esac
}

ntfyTopic="$(getTopic "$taskName")"

# ===================== TITLE =====================
getTitle() {
  case "$1" in
    downloadPodcast*) echo "Podcast Descarregat"  ;;
    moveBooks)        echo "Nou Llibre"           ;;
    moveMusic)        echo "Música Descarregada"  ;;
    moveMediaMovies)  echo "Nova Pel·lícula"      ;;
    moveMediaSeries)  echo "Nou Capítol"          ;;
    *)                echo "Acció desconeguda"    ;;
  esac
}

title="$(getTitle "$taskName")"

# ===================== BODY =====================
ntfyBody="​
📌 Titol: ${titol}"

if [[ -n "$autor" ]]; then
  ntfyBody="${ntfyBody}"$'\n'"🎤 ${autor}"
fi

if [[ -n "$contentSize" ]]; then
  ntfyBody="${ntfyBody}"$'\n'"💾 Tamany: ${contentSize}"
fi

# ===================== SEND =====================
curlResponse=$(curl -s \
  -w "\n%{http_code}" \
  --max-time 10 \
  --connect-timeout 5 \
  -H "Authorization: Bearer ${ntfyToken}" \
  -H "Title: ${title}" \
  -H "Tags: ${tags}" \
  -H "Priority: default" \
  -H "Content-Type: text/plain" \
  -d "${ntfyBody}" \
  "${ntfyServer}/${ntfyTopic}")

httpCode=$(echo "$curlResponse" | tail -n1)
curlBody=$(echo "$curlResponse" | head -n-1)

# ===================== LOG =====================
logFile="${SCRIPT_DIR}/sendNtfy.log"
maxSize=102400  # 100 KB

if [ -f "$logFile" ]; then
  currentSize=$(stat -c%s "$logFile" 2>/dev/null || wc -c < "$logFile")
  if [ "$currentSize" -gt "$maxSize" ]; then
    : > "$logFile"
  fi
fi

echo "$(date '+%Y-%m-%d %H:%M:%S') [DEBUG] task=${taskName} | titol=${titol}" >> "$logFile"

if [ "$httpCode" != "200" ] && [ "$httpCode" != "202" ]; then
  echo "$(date '+%Y-%m-%d %H:%M:%S') [ERROR] HTTP ${httpCode} | response=${curlBody}" >> "$logFile"
  exit 1
fi

echo "$(date '+%Y-%m-%d %H:%M:%S') [OK] Sent ${taskName}" >> "$logFile"

Como podéis ver, dependiendo de la tarea (task) se envía un título y un mensaje correspondiente.

Además, también mantengo un pequeño archivo log por si se produce algún error y necesito revisar el motivo.
